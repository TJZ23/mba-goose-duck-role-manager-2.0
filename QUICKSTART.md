# 快速开始

欢迎使用《鹅鸭杀：大厅潜行版》工委控制台 - Socket.io 实时同步版本! 🎮

这个版本已经从 PeerJS 点对点模式升级为基于**真实网络 Socket.io** 的版本，支持任意距离的跨网络实时同步。

## 🚀 30 秒快速开始

### 1️⃣ 安装依赖

```bash
cd goose-duck-role-manager
npm install
```

### 2️⃣ 启动服务器

```bash
npm start
```

看到这条消息说明成功：
```
[服务器] 《鹅鸭杀控制台》服务器运行在 http://localhost:3000
```

### 3️⃣ 打开浏览器

```
http://localhost:3000
```

## ✨ 核心特性

### ✅ 独立操作意识
- 每个用户可以**独立操作**，UI 立即响应
- 不需要等待服务器确认

### ✅ 实时状态同步  
- 任何修改在 **0-500ms** 内同步给房间内所有用户
- 多个用户在不同地方同时修改，不会相互阻塞

### ✅ 跨网络通讯
- 基于 Socket.io，支持**任意距离**的网络通讯
- 不受 NAT、防火墙等限制（使用中央服务器中转）

## 🧪 快速测试 - 两个窗口同步

1. **打开两个浏览器窗口**

   窗口 1: `http://localhost:3000`
   
   窗口 2: `http://localhost:3000`

2. **窗口 1 中分享房间号**
   - 记住显示在"房间："后面的 6 位房间号（如 `123456`）

3. **窗口 2 中加入房间**
   - 点击"🔗 加入房间"
   - 输入房间号
   - 点击"🚀 直接加入"

4. **开始同步测试**
   - 在窗口 1 中修改游戏数据（如添加玩家、配置队伍）
   - 观察窗口 2 是否立即同步显示相同内容
   - 互换角色：在窗口 2 修改，观察窗口 1 是否同步

   ✅ 成功！你已经体验到了实时同步的魔力

## 💡 工作原理

### Socket.io 架构

```
┌────────────────────────────────────────────┐
│          Socket.io 服务器                   │
│         (Node.js + Express)                │
│                                            │
│  • 房间管理: 维护房间和用户列表            │
│  • 状态转发: 广播所有操作和状态更新        │
│  • 连接管理: 处理连接/断开                  │
└────────────┬────────────────────┬──────────┘
             │                    │
      ┌──────▼────────┐   ┌──────▼──────────┐
      │  用户 A       │   │  用户 B         │
      │  (房主)        │   │  (客户端)       │
      │  • 本地操作    │   │  • 本地操作     │
      │  • 立即反馈    │   │  • 立即反馈     │
      │  • 同步结果    │   │  • 同步结果     │
      └──────┬────────┘   └──────┬──────────┘
             │                    │
             └────────────────────┘
          所有操作相互同步
```

### 事件流

```
用户操作 → 本地立即反馈 → 发送到服务器 → 广播给房间内所有人 → 其他人看到相同结果
  ∟─────────乐观更新────────────确保最终一致─────────────────∘
```

## ❓ 常见问题

**Q: 我能和朋友在不同城市一起玩吗？**

A: ✅ 可以！请参考[部署指南](./DEPLOY.md)将服务器部署到阿里云、AWS 等云平台，然后分享服务器地址给朋友。

**Q: 本地测试时延迟多少？**

A: 通常 **50-200ms**，这是网络往返时间。由于使用了乐观更新（立即反馈），用户感受不到延迟。

**Q: 可以同时让多少人加入？**

A: 单个服务器理论上支持数千个并发连接。具体取决于服务器配置和网络带宽。

**Q: 如果中途掉线了怎么办？**

A: Socket.io 会自动尝试重新连接（最多 10 次）。重连成功后会同步最新的房间状态。

**Q: 数据会被保存吗？**

A: 目前存储在浏览器本地 localStorage（每 20 秒保存一次）。升级版本可以配置数据库持久化。
