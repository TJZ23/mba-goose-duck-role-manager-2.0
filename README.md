# 《鹅鸭杀：大厅潜行版》工委控制台 - Socket.io 实时同步版本

## 概览

这是一个基于 **Socket.io** 的真正跨网络实时同步版本，支持多个用户在不同地方秒级同步游戏状态。

### 核心特性 ✨

- ✅ **真实跨网络同步**：基于 Socket.io，支持任意距离的跨网络通讯
- ✅ **独立操作意识**：每个用户都有各自的独立操作意识，UI 立即响应自己的操作
- ✅ **实时状态同步**：所有修改和操作结果在 0-500ms 内同步给房间内的其他用户
- ✅ **自动保存**：状态每 20 秒自动保存，并实时广播同步
- ✅ **房间历史**：自动记录最近加入的 8 个房间，快速重连

## 快速开始

### 1. 安装依赖

```bash
cd goose-duck-role-manager
npm install
```

### 2. 启动服务器

```bash
npm start
```

或使用开发模式（自动重启）：

```bash
npm run dev
```

服务器将运行在 `http://localhost:3000`

### 3. 访问应用

在浏览器中打开：`http://localhost:3000`

## 使用说明

### 创建房间

1. 打开应用后，您会获得一个**房间号**（6 位数字）
2. 将此房间号分享给队友
3. 队友在"加入房间"中输入房间号即可加入

### 加入房间

1. 点击"🔗 加入房间"按钮
2. 在弹窗中输入房间号或从历史记录中选择
3. 点击"🚀 直接加入"

## 实时同步工作原理

```
用户 A (独立操作)        服务器                   用户 B (独立操作)
  • 点击按钮      →  接收操作  →  广播给房间  →   • 自动同步 UI
  • 立即反馈          更新状态      (0-500ms)        • 看到相同结果
```

**关键特性：**
- 用户点击按钮后，**本地 UI 立即响应**（不等待服务器）
- 操作通过 Socket.io 转发，确保房间内所有人都收到
- 由于使用单一 Socket 连接，自然避免了日志条件和并发冲突

## 网络事件

### 事件列表

| 事件 | 方向 | 说明 |
|------|------|------|
| `join-room` | 客户端 → 服务器 | 加入房间 |
| `update-state` | 客户端 → 服务器 | 更新游戏状态 |
| `user-action` | 客户端 → 服务器 | 用户操作 |
| `leave-room` | 客户端 → 服务器 | 离开房间 |
| `state-updated` | 服务器 → 客户端 | 状态已更新 |
| `user-action` | 服务器 → 客户端 | 接收其他用户的操作 |
| `user-joined` | 服务器 → 客户端 | 有新用户加入 |
| `user-left` | 服务器 → 客户端 | 有用户离开 |

## 常见问题

### Q: 我能和朋友在不同城市一起玩吗？

A: 可以！请部署服务器到云平台（如 AWS、阿里云等），然后分享服务器地址给朋友。详见 DEPLOY.md。

### Q: 本地测试时延迟是多少？

A: 通常 **50-200ms**。由于使用乐观更新（本地立即反馈），用户感受不到延迟。

### Q: 如果掉线了会怎样？

A: Socket.io 会自动尝试重新连接（最多 10 次），重连成功后会同步最新的房间状态。
