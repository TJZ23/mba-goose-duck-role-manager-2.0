# 《鹅鸭杀：大厅潜行版》工委控制台 - Socket.io 实时同步版本

## 概览

这是一个基于 **Socket.io** 的真正跨网络实时同步版本，支持多个用户在不同地方秒级同步游戏状态。

### 新特性 ✨

- ✅ **真实跨网络同步**：基于 Socket.io，支持任意距离的跨网络通讯
- ✅ **独立操作意识**：每个用户都有各自的独立操作意识，UI 立即响应自己的操作
- ✅ **实时状态同步**：所有修改和操作结果在 0-500ms 内同步给房间内的其他用户
- ✅ **无需房主在线**：服务器端房间管理，房主和客户端地位对等
- ✅ **自动保存**：状态每 20 秒自动保存，并实时广播同步
- ✅ **房间历史**：自动记录最近加入的 8 个房间，快速重连

## 使用说明

### 创建房间

1. 打开应用后，您会获得一个**房间号**（6 位数字）
2. 将此房间号分享给队友
3. 队友在"加入房间"中输入房间号即可加入

### 加入房间

1. 点击"🔗 加入房间"按钮
2. 在弹窗中输入房间号或从历史记录中选择
3. 点击"🚀 直接加入"

### 实时同步如何工作

```
┌─────────────────────────────────────────────────────────┐
│                     Socket.io 服务器                      │
│                    (房间状态管理)                         │
└──────────────┬──────────────────────────┬────────────────┘
               │                          │
      ┌────────▼────────┐      ┌─────────▼──────────┐
      │  用户 A (房主)   │      │  用户 B (客户端)    │
      │   独立操作意识   │      │   独立操作意识      │
      │  • 点击按钮      │      │  • 点击按钮        │
      │  • 修改参数      │      │  • 修改参数        │
      │  • 立即反馈      │      │  • 立即反馈        │
      │                 │      │                    │
      │  操作发送给服务器│      │  操作发送给服务器   │
      └────────┬────────┘      └─────────┬──────────┘
               │                          │
               └──────────────┬───────────┘
                              │
                    ┌─────────▼─────────┐
                    │ 服务器接收和处理   │
                    │ 广播给房间内所有人 │
                    └────────┬──────────┘
                             │
                    ┌────────▼─────────┐
                    │ 所有客户端接收     │
                    │ 同步本地 UI 状态   │
                    └──────────────────┘
```

**关键特性**：
- **快速响应**：用户点击按钮后，本地 UI 立即响应（不等待服务器确认）
- **可靠同步**：操作通过 Socket.io 转发，确保房间内所有人都收到更新
- **冲突规避**：由于 Socket.io 是基于事件的单一连接，自然避免了并发冲突

## 网络架构

### 后端（server.js）

- **Express 服务器**：提供静态文件和 HTTP 路由
- **Socket.io 服务器**：处理实时通讯
- **房间管理**：维护房间和用户列表
- **状态转发**：广播用户操作和状态更新

### 前端（index.html）

- **Socket.io 客户端**：连接到服务器
- **本地状态管理**：维护本地游戏状态
- **操作拦截**：拦截用户输入，发送给服务器
- **远程操作重放**：接收并执行来自其他用户的操作

## 事件流

### 1. 用户加入房间

```
客户端: socket.emit('join-room', { roomId, userId })
  ↓
服务器: 记录用户，创建或更新房间
  ↓
服务器: socket.emit('room-joined', { roomId, users })
  ↓
客户端: 更新 UI，显示房间号和用户列表
```

### 2. 用户操作（如点击按钮）

```
用户 A: 点击按钮 "✅ +1 完成任务"
  ↓
本地 UI: 立即更新（乐观更新）
  ↓
发送操作: socket.emit('user-action', { roomId, action, payload })
  ↓
服务器: 接收操作，广播给房间内所有人
  ↓
所有客户端: 收到 socket.on('user-action')
  ↓
其他用户 B 的 UI: 同步更新
```

### 3. 状态同步

```
自动保存: 每 20 秒或手动保存
  ↓
发送状态: socket.emit('update-state', { roomId, state })
  ↓
服务器: 更新房间的游戏状态
  ↓
所有客户端: 收到 socket.on('state-updated')
  ↓
本地状态: 与远程状态合并（调和）
```

## 常见问题

### Q: 如何在公网上部署？

A: 
1. 将 `server.js` 部署到服务器（如 Heroku、AWS、阿里云等）
2. 在客户端代码中修改连接地址为服务器地址
3. 确保服务器防火墙允许 WebSocket 连接（3000 端口或您配置的端口）

### Q: 如何保证数据安全？

A:
- 建议在生产环境使用 HTTPS/WSS（加密连接）
- 可以添加房间密码或授权机制（修改 `server.js`）
- 定期备份游戏状态到数据库

### Q: 支持多少个并发用户？

A:
- 单个服务器实例可支持数千个并发连接
- 对于大规模部署，可以使用 Redis 适配器实现多服务器房间共享
- 详见 Socket.io 官方文档的 Redis 适配器部分

### Q: 如果网络中断会怎样？

A:
- Socket.io 会自动尝试重新连接（配置为最多 10 次）
- 重连成功后，客户端会接收最新的房间状态
- 本地状态在 localStorage 中保存，不会丢失

## 故障排除

### 连接失败

1. 检查服务器是否正常运行：`npm start`
2. 检查客户端地址是否正确（默认为 localhost:3000）
3. 检查防火墙是否阻止 3000 端口
4. 查看浏览器控制台的错误信息

### 状态不同步

1. 刷新页面强制重新同步
2. 检查网络延迟（浏览器 DevTools > Network）
3. 清除 localStorage 重试：`localStorage.clear()`

### 房间消失

- 房间在所有用户离开 30 秒后会自动删除
- 刷新页面重新创建房间

## 开发指南

### 添加新的网络事件

在 `server.js` 中：

```javascript
socket.on('custom-event', (data) => {
  io.to(data.roomId).emit('custom-event-response', {
    // 处理逻辑
  });
});
```

在 `index.html` 中：

```javascript
socket.on('custom-event-response', (data) => {
  // 处理响应
});
```

### 添加持久化存储

修改 `server.js`，使用数据库：

```javascript
import sqlite3 from 'sqlite3';
const db = new sqlite3.Database('./rooms.db');

// 保存房间状态
db.run('INSERT INTO rooms VALUES (?, ?)', [roomId, JSON.stringify(state)]);
```
